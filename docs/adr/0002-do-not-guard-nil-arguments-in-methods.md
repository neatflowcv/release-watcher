# ADR-0002: 메서드에서 nil 인자 가드를 두지 않는다

## 상태

Accepted

## 배경

도메인 객체나 저장소 구현 메서드는 내부 로직에서 필요한 객체를 전제로 동작한다.
호출자가 `nil`을 넘기는 상황은 정상적인 비즈니스 오류라기보다 프로그래밍 오류에 가깝다.

예를 들어 `CreateProject`에 `nil` 프로젝트를 넘기는 것은 저장 실패로 복구할 수 있는
상황이 아니라 호출부 계약 위반이다.

## 결정

메서드에서 `nil` 인자 방어 코드를 두지 않는다.

## 근거

- `nil` 인자는 호출부의 프로그래밍 오류로 본다.
- 프로그래밍 오류를 일반 `error`로 감싸면 문제 위치가 흐려진다.
- 필요한 객체는 메서드 안에서 바로 사용하고, 잘못된 호출은 자연스럽게 panic이 나도록 둔다.
- 저장소 메서드는 정상적인 외부 실패만 `error`로 반환한다. 예를 들면 데이터베이스 실행
  실패, 조회 실패, row scan 실패가 여기에 해당한다.

## 결과

- `CreateProject`는 `nil` 프로젝트를 별도 검사하지 않는다.
- `nil` 프로젝트가 넘어오면 프로젝트 메서드 호출 과정에서 panic이 발생한다.
- 생성자에서 구성 요소를 검증하는 것은 이 결정의 범위 밖이다.
