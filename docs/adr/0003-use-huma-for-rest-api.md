# ADR-0003: REST API 구현에 `huma`를 사용한다

## 상태

Accepted

## 배경

release-watcher는 REST API를 제공해야 한다.

API 구현 라이브러리는 Go 코드만으로 다룰 수 있고, 요청과 응답을 구조체로 정의할 수
있는 방식이 적합하다.

## 결정

REST API 구현에 `huma`를 사용한다.

## 근거

- Go 코드만 있으면 API를 정의하고 구현할 수 있다.
- 별도 스펙 파일이나 코드 생성부터 시작하지 않아도 된다.
- 요청 입력과 응답 출력을 구조체로 정의할 수 있다.
- 핸들러는 구조체 입력을 받고 구조체 출력을 반환하는 형태로 작성할 수 있다.

## 결과

- REST API 계층은 `huma` 기반으로 작성한다.
- API 요청/응답 DTO는 구조체로 정의한다.
- OpenAPI 스펙 선작성이나 codegen 기반 구현은 기본 방향으로 보지 않는다.
